# 📖 第一部分：项目背景与目标

## 1.1 项目背景

### 传统声乐学习的困境

在传统声乐学习中，学习者面临着诸多挑战：

| 困境 | 描述 | 影响 |
|------|------|------|
| 缺乏实时反馈 | 只能在课后获得教师点评，无法实时了解自己的演唱问题 | 学习效率低 |
| 反馈主观性强 | 教师评价因人而异，缺乏客观量化的评估标准 | 进步难以衡量 |
| 练习成本高昂 | 专业声乐课程费用昂贵，时间安排不灵活 | 学习门槛高 |
| 缺乏针对性指导 | 无法针对个人问题提供个性化练习建议 | 提升缓慢 |

### 人工智能时代的机遇

深度学习和神经网络技术的快速发展，为声乐教育带来了新的可能性：

- **语音识别技术** → 精准捕捉演唱音高
- **信号处理技术** → 实时分析音频特征
- **自然语言处理** → 智能生成点评建议
- **计算机视觉** → 可视化音高曲线对比

## 1.2 项目目标

本研究项目旨在构建一个**基于AI的智能K歌声乐训练平台**，通过结合先进的深度学习算法和现代Web技术，为用户提供：

1. **高精度音高检测** - 基于深度学习模型实现误差小于5音分的音高提取
2. **实时可视化反馈** - 实时显示演唱音高曲线，与参考音高进行对比
3. **AI智能点评** - 从多个维度对演唱进行专业评估
4. **个性化改进建议** - 基于AI分析提供针对性的练习方向

### 传统方法 vs 深度学习方法

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 自相关 (Autocorrelation) | 找信号与自身的相关性峰值 | 简单快速 | 对噪声敏感 |

| **Swift-F0 (本项目)** | **深度神经网络** | **精度高抗噪强** | **需要训练** |

### 神经网络结构详解

Swift-F0采用**深度神经网络（DNN）**架构，核心组件包括：

**1. 输入层**
```
输入: 513维频谱向量 (STFT后)
或:  40维MFCC特征向量
```

**2. 隐藏层**
```
Layer 1: 512神经元 + ReLU + Dropout(0.2)
Layer 2: 512神经元 + ReLU + Dropout(0.2)
Layer 3: 256神经元 + ReLU + Dropout(0.2)
残差连接: 输入 → Layer 3输出 (跳跃连接)
```

**3. 输出层**
```
输出: 帧级别的基频概率 (Sigmoid)
      后处理: 维特比解码确保时序连续性
```

## 3.3 音高评估与评分算法

### 音分（Cents）计算

音高偏差使用**音分（Cents）**作为度量单位，这是音乐学中的标准单位：

```
cents = 1200 × log₂(user_pitch / ref_pitch)
```

```python
def calculate_cents(user_pitch: float, ref_pitch: float) -> float:
    """
    计算用户音高与参考音高的音分偏差

    参数:
        user_pitch: 用户演唱的音高 (Hz)
        ref_pitch: 参考音高 (Hz)

    返回:
        cents: 音分偏差
            0音分 = 完全一致
            ±50音分 = 约半音
            ±100音分 = 约一个全音

    示例:
        >>> calculate_cents(440, 442)  # A4 vs 略高的A4
        -7.851  # 约-8音分
    """
    if user_pitch <= 0 or ref_pitch <= 0:
        return 0  # 无效音高

    cents = 1200 * math.log2(user_pitch / ref_pitch)
    return cents
```

### 评分计算

综合评分由多个维度加权计算：

```python
def calculate_overall_score(
    pitch_score: float,      # 音高准确度 (0-100)
    rhythm_score: float,    # 节奏稳定性 (0-100)
    stability_score: float, # 音高稳定性 (0-100)
    weights: dict = {'pitch': 0.5, 'rhythm': 0.3, 'stability': 0.2}
) -> float:
    """
    计算综合评分

    权重可根据不同歌曲类型或用户偏好进行调整
    """
    overall = (
        weights['pitch'] * pitch_score +
        weights['rhythm'] * rhythm_score +
        weights['stability'] * stability_score
    )
    return round(overall, 1)
```

### 亮点四：多维度AI评估体系

**创新描述**：基于深度学习的全方位演唱评估

**评估模型**：
```python
class VocalCritiqueModel(nn.Module):
    def __init__(self):
        super().__init__()
        # CNN特征提取
        self.cnn = nn.Sequential(
            nn.Conv2d(1, 64, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(64, 128, 3, padding=1),
            nn.ReLU(),
        )

        # Transformer时序建模
        self.transformer = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(d_model=128, nhead=8),
            num_layers=3
        )

        # 多标签输出
        self.fc = nn.Linear(128, 5)  # 5个评估维度

    def forward(self, x):
        # x: (batch, 1, 40, time_steps)
        x = self.cnn(x)
        x = x.permute(2, 0, 1)  # (time, batch, features)
        x = self.transformer(x)
        x = x.mean(dim=0)  # 全局平均池化
        return torch.sigmoid(self.fc(x))  # 0-1概率输出
```
